# 西湖景区实时客流监控看板

一个基于高德地图和大语言模型的智能景区客流监控系统，提供实时客流监控、客流预测预警、智能路线规划和AI对话导游功能。支持阿里云百炼（Qwen系列）和OpenAI双模型接入。

## 🌟 功能特性

### 1. 实时客流监控与地图看板
- **15个景点全覆盖**：断桥、苏堤、雷峰塔、灵隐寺、三潭印月、湖滨、太子湾、曲院风荷、白堤、花港观鱼、岳王庙、柳浪闻莺、平湖秋月、孤山、西泠印社
- **实时数据刷新**：每8秒自动更新人流数据
- **动态地图标记**：标记点大小和颜色随拥挤度实时变化
- **四级拥挤度**：空闲（蓝）、舒适（绿）、较挤（黄）、拥挤（红）
- **顶部统计卡片**：总游客数、拥挤景点数、畅通景点数
- **历史趋势图**：点击景点查看过去24小时人流变化曲线
- **状态变化指示**：相比1小时前的人流变化趋势和百分比
- **详情面板**：今日峰值、当前排名、建议游览时段、智能游览建议

### 2. 客流预测分析与预警
- **未来2小时预测**：每10分钟一个采样点，共13个预测点
- **智能预测算法**：基于时段规律 + 景点热度权重 + 随机波动
- **峰值自动检测**：识别预测时段内的客流高峰
- **AI预警文案**：大模型生成专业、友好的预警建议
- **可视化展示**：ECharts折线图展示预测趋势，标注峰值点

### 3. 智能路线规划
- **智能选路策略**：优先选择低人流景点作为途经点
- **避开拥挤区域**：自动过滤高饱和度景点
- **路线可视化**：地图上高亮显示推荐路线（带方向箭头）
- **自动缩放视角**：自动调整地图视角以展示完整路线
- **AI导游解说**：大模型生成个性化路线指引和游览建议

### 4. 智能对话交互
- **多意图识别**：预测、路线规划、帮助、问候、未知意图
- **景点模糊匹配**：支持全称和关键词模糊匹配
- **多种指令格式**：
  - 客流预测：`预测断桥`、`雷峰塔人多吗`
  - 路线规划：`路线 断桥 雷峰塔`、`从灵隐寺到湖滨怎么走`、`规划 苏堤 白堤`
- **快捷操作按钮**：一键触发常用功能
- **打字指示器**：模拟AI思考效果
- **对话历史**：保留完整对话记录

## 🚀 快速开始

### 环境要求

- Node.js >= 18.0.0
- npm >= 9.0.0

### 安装依赖

```bash
cd backend
npm install
```

### 配置环境变量

复制 `.env.example` 为 `.env` 并填入你的密钥：

```bash
cp .env.example .env
```

编辑 `.env` 文件：

```env
# 服务端口
PORT=3000

# 高德地图API密钥（前端已内置，可选配置）
AMAP_KEY=your_amap_key_here

# LLM服务商选择：dashscope（阿里云百炼）或 openai
LLM_PROVIDER=dashscope

# ========== 阿里云百炼配置 ==========
DASHSCOPE_API_KEY=your_dashscope_api_key_here
DASHSCOPE_MODEL=qwen3.6-flash
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1

# ========== OpenAI配置（可选） ==========
OPENAI_API_KEY=your_api_key_here
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-3.5-turbo
```

#### 获取API密钥

**阿里云百炼（推荐）**：
1. 访问 [阿里云百炼控制台](https://bailian.console.aliyun.com/)
2. 注册/登录阿里云账号
3. 进入"模型服务" → "API密钥管理"
4. 创建API密钥（格式：`sk-xxxxxxxxxxxxxxxx`）

**OpenAI**：
1. 访问 [OpenAI Platform](https://platform.openai.com/)
2. 注册/登录账号
3. 进入 API Keys 页面
4. 创建新的密钥

### 启动服务

```bash
# 开发模式
npm run dev

# 生产模式
npm start
```

启动成功后访问：`http://localhost:3000/xihu-dashboard.html`

## 📊 算法说明

### 时间因子计算

根据当前小时计算全天人流基准系数：

| 时间段 | 系数 | 说明 |
|--------|------|------|
| 06:00-08:00 | 0.15 | 清晨低峰 |
| 08:00-10:00 | 0.35 | 早间上升 |
| 10:00-12:00 | 0.55 | 上午高峰 |
| 12:00-14:00 | 0.70 | 午间高峰 |
| 14:00-17:00 | 0.90 | 下午最高峰 |
| 17:00-19:00 | 0.70 | 傍晚回落 |
| 19:00-21:00 | 0.40 | 夜间下降 |
| 21:00-23:00 | 0.15 | 夜间低峰 |
| 其他时段 | 0.05 | 深夜几乎无人 |

### 拥挤度等级

| 饱和度范围 | 状态 | 颜色 | 等级 |
|-----------|------|------|------|
| < 20% | 空闲 | #1890ff（蓝） | 0 |
| 20%-40% | 舒适 | #52c41a（绿） | 1 |
| 40%-67% | 较挤 | #faad14（黄） | 2 |
| > 67% | 拥挤 | #ff4d4f（红） | 3 |

### 实时人流模拟算法

```
人流 = 容量 × 时间因子 × 景点热度 × (0.85 + random(0, 0.30))
```

### 预测算法

```
预测值(t) = 容量 × 时段系数(hour) × 景点热度 × (0.85 + random(0, 0.30))
```

### 路线规划策略

1. 按人流升序排列所有景点
2. 取前5个低人流景点作为候选中间节点
3. 过滤掉起点和终点
4. 从候选中选择2个作为途经点
5. 构建完整路径：起点 → 途经点1 → 途经点2 → 终点

### Prompt模板

**客流预警Prompt**：
```
你是一个景区客流分析专家。根据以下数据生成一段简洁的预警文字：
- 景点名称：{景点名}
- 预测未来2小时峰值：{峰值}人（饱和度{百分比}%）
- 峰值预计出现时间：{时间点}
- 当前人流：{当前人数}

要求：
1. 语气专业且友善
2. 包含具体数字
3. 给出明确建议
4. 控制在60字以内
```

**路线规划Prompt**：
```
你是一个景区导游。根据以下路线和人流数据，生成一段友好的路线指引：
- 起点：{起点名}（当前人流{人数}人）
- 途经1：{景点名}（当前人流{人数}人）
- 途经2：{景点名}（当前人流{人数}人）
- 终点：{终点名}（当前人流{人数}人）

要求：
1. 说明选择此路线的原因（避开拥挤）
2. 给出每个节点的预计游览建议
3. 语气亲切，控制在80字以内
```

### 降级策略

当大模型API不可用（未配置密钥、网络错误等）时，系统自动使用本地生成的备用文案，确保功能不中断。
